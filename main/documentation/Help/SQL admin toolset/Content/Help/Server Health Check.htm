Server Health Check

The Server Health Check tool helps you keep a constant eye on the status of SQL Server instances across your enterprise. You can run Server Health Check as centralized interface or as system tray service alerts you when instance goes offline or becomes unhealthy.

What does Server Health Check offer?
  • Monitor a SQL Server instance or a server group
  • Run tool in system tray
    • Use status icon to quickly see health of the instance
    • Use pop-up alerts to monitor when instances come online or go offline
    • Launch tool on Windows startup
  • Refresh data automatically at specified intervals
  • Start, stop, pause, resume, and restart servers
  • Avoid unnecessary alerts by selecting which instances should be ignored during maintenance periods
  • Choose WMI or a SQL query to collect health information
 How do I use Server Health Check?
  1. Specify the SQL Server instance or server group that contains the logins whose passwords you want to check.To specify more than one instance, complete the following steps:
    1. Click Add.
    2. On the Add Server window, type the name of each instance, separating multiple instances with semicolons.
    3. Specify the type of authentication you want Server Health Check to use to connect to the specified instances. By default, the credentials of your current Windows account are used.
    4. To verify that the connection settings are valid for the specified instances, click Test Connection.
    5. Indicate whether you want to receive overall status information or alerts and notifications about the specified instance.
    6. Click OK.
  2. Select the appropriate system tray options.
  3. Choose whether you to run a WMI or SQL query to retrieve the status of the specified instances. You can specify which SQL command you want to run in the query.
  4. ClickCheck Health Now.
 How does Server Health Check work?

By default, Server Health Checker connects to the specified SQL Server instances using the credentials of your currently logged on Windows account.

Server Health Check then uses either a WMI or SQL query to collect information, such as whether the SQL Server services are running, and when an instance comes online or goes offline.
