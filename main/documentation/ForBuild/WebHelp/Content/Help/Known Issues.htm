Known issues

Idera strives to ensure our products provide quality solutions for your SQL Server needs. We are currently researching the following known issues. If you need further assistance with any issue, please contact Support (www.idera.com/support).

Installation and configuration considerations

SQL admin toolset no longer supports Itanium

Idera discontinued Itanium support in SQL admin toolset version 1.6 and later. However, versions 1.5 and earlier continue to operate with Itanium, but support for those versions is no longer available. For more information, see the product requirements.

SQL admin toolset uses the administrator login by default on Windows Vista

Installing the SQL admin toolset in Windows Vista requires administrator credentials. If you launch the SQL admin toolset at the end of the installation, the product runs with that user login. Any settings saved by the user during that session are lost during the next restart, and it launches with a regular user logged in.

Known issues in this version

There are no new known issues in this version.

Previous known issues

Linked Server Copy: does not support local computer accounts

If you create a linked server and map the security locally, the linked server is created but you receive an error stating that it is not a valid login.

Database Mover: Copying large database may require additional time and resources

Database Mover may experience performance issues when copying a large database of 10GB or greater to another server.

Server Configuration: Limited keyboard browsing is available

When browsing multiple servers, you can highlight a server using the directional arrow keys but cannot update the server details. Press Enter to refresh the screen and display the correct server information.
