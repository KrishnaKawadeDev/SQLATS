Table Pin

The Table Pin tool allows you to manage pinned SQL Server tables. Pinning tables into SQL Server memory can provide performance boosts when SQL Server accesses those tables. Despite the performance benefit, this feature can become dangerous and cause serious performance issues when misused. With this tool, you can:

  • Monitor exactly which tables are pinned and how much memory they consume from the SQL Server memory pool
  • Pin or unpin tables and easily set up the proper configuration

This tool only works for SQL Server 2000 instances since the Pinned Table feature was removed in SQL Server 2005 and later.

By default, Table Pin connects to the specified SQL Server instances using the credentials of your currently logged on Windows account, and then determines which database tables are pinned to the SQL Server memory.

To pin a table:

  1. Specify the SQL Server whose table status you want to view.
  2. Clear the Hide tables that are not pinned option.
  3. Click Get Pinned Status.
  4. Select the table you want to pin, and then click Pin Table. The action takes effect immediately.

To view a pinned status:

  1. Specify the SQL Server whose table status you want to view. If you want to see the status of all tables on this SQL Server instance, clear the Hide tables that are not pinned option.
  2. Click Get Pinned Status.
  3. If you want to save the results, copy the list to clipboard, or export the list as an XML or CSV file.

To unpin a table:

  1. Specify the SQL Server whose table status you want to view. 
  2. Click Get Pinned Status.
  3. Select the table you want to unpin, and then click Unpin Table. The action takes effect immediately.
